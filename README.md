# Bricola.md

ブラウザだけで動く、オフライン・単一フォルダ配布の Markdown ビューア／エディタ。サーバ不要。`index.html` を `file://` で開き、[File System Access API](https://developer.mozilla.org/docs/Web/API/File_System_API) で選んだフォルダ（リポジトリ）内の Markdown を、パス指定で開いて閲覧・編集・保存する。

## 名前の由来

レヴィ=ストロースの **ブリコラージュ**（ありあわせの道具・材料を寄せ集めて作ること）から。ビルドもサーバも持たず、ブラウザにある File System Access API と同梱ライブラリだけで組み上げる作り方にちなむ。

## 特徴

- **サーバ不要・オフライン** — 依存ライブラリを同梱。`index.html` をダブルクリック（`file://`）で起動。
- **リポジトリ選択** — 起動後にフォルダを選ぶ。アプリ（`bricola-md/`）の置き場所は対象リポジトリと独立でよい。設定ファイルは不要。
- **パス指定で開く** — パスバーにリポジトリ相対パスを入力して開く。ディレクトリ指定はその直下 md をサイドバーに展開、md 指定はその 1 枚を表示。指定したディレクトリ／ファイルはサイドバーに蓄積される。
- **シェル流の TAB 補完** — 入力するたび前方一致候補を表示。TAB で候補が 1 つなら確定（ディレクトリは続けて中へ降りる）、複数なら順送り。全木は走査せず必要な階層だけ読むので起動が軽い。
- **閲覧と編集** — ソース／プレビューの 2 モード。プレビューは marked + DOMPurify。
- **安全な保存** — 最小差分・原形保持（改行や frontmatter を温存）。保存・再読込時に競合をガード。
- **相対画像** — md からの相対パス画像を FSA 経由で Blob URL として解決。
- **差し込み** — サイドバーのファイルを D&D で本文の指定位置に実体貼付（コピー）。
- **アウトライン表示・テーブル編集**。

## 使い方

1. `bricola-md/` を任意の場所に置く。
2. `index.html` をブラウザで開く（`file://` 可。File System Access API 対応の Chromium 系ブラウザが必要）。
3. 「リポジトリを選択」で、Markdown を含むフォルダを選ぶ。
4. パスバーにリポジトリ相対パスを入力して開く（下記）。
5. 閲覧・編集し、保存する。

## パス指定と補完

パスバーにリポジトリ相対パスを入力し、Enter または「開く」で開く。種別は FS 上で自動判定する（末尾スラッシュ不要）。

- **ディレクトリ** → サイドバーにそのディレクトリのノードを追加し、**直下の md のみ**を展開表示（再帰しない）。
- **md ファイル** → その md だけをサイドバーに足して中央に表示。
- 指定するたびにサイドバーへ**蓄積**される（同じ場所は重複させない・各ノードは × で個別削除）。リロードしても直前の状態を復元する。

入力中の補完はシェル流:

- 入力するたびに**前方一致候補**をドロップダウン表示し、最有力候補の後方をインライン補完する。
- **TAB**: 候補が 1 つならそのまま確定（ディレクトリは末尾 `/` を付け、続けて TAB で中へ降りる）。複数なら順送りで、入力欄とドロップダウンの選択が連動する。
- **Enter**: md ファイルなら開く。ディレクトリ（末尾 `/`）なら開かずに中へ降りる。ディレクトリ自体を開く（直下 md を一覧化する）には「開く」ボタンを使う。
- 候補はディレクトリと `.md` のみ。必要な階層だけを 1 階層ずつ読むため、大規模リポジトリでも起動・補完が軽い。

## 制約

- File System Access API 対応ブラウザが必要（Chromium 系）。`file://` でも動作する。
- v1 は既存 md の**閲覧・編集・保存のみ**。新規作成・削除・リネームは行わない。
- 補完候補に `node_modules` / `.git` / ドットファイルも含まれる（除外設定は持たない）。
- 表示はパス／名前順。表示名の上書きや並び替えは未対応。

## 構成

```
bricola-md/
  index.html      エントリ。同梱ライブラリと app/*.js を classic script で読み込む
  app/            アプリ本体（Bricola 名前空間の classic script 群。pathbar.js が補完を担う）
  styles/         スタイル
  vendor/         同梱ライブラリ（marked / DOMPurify / turndown / mermaid）
```

設計の詳細は [`../docs/`](../docs/)（DESIGN / PLAN / ADR / 用語集）を参照。パス指定ロードへの転換は [ADR-0014](../docs/adr/0014-path-driven-loading-and-directory-nodes.md)。
